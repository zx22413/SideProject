# 🔐 敏感文件同步指南

> **問題**：Git 忽略了包含 Token 的 `.gs` 和 `.csv` 文件，換電腦後如何同步？

---

## 🎯 推薦方案（依安全性排序）

### **方案 1：配置文件分離（最安全）** ⭐⭐⭐

**原理**：將敏感資訊抽離為配置文件，只同步模板

**步驟**：

#### **首次設置（當前電腦）**
1. 已有 `config.template.gs` 模板文件（會推送到 Git）
2. 複製並重命名：`config.template.gs` → `config.gs`
3. 在 `config.gs` 中填入真實的 Token 和 Sheets ID
4. 修改主程式碼，從 `config.gs` 讀取配置

#### **換電腦後（家裡電腦）**
1. 從 Git 拉取最新代碼（包含 `config.template.gs`）
2. 複製 `config.template.gs` → `config.gs`
3. 填入敏感資訊（從 LINE Developers 和 Google Sheets 獲取）
4. 完成！`config.gs` 會被 `.gitignore` 自動忽略

**優點**：
- ✅ 最安全（敏感資訊永不離開本地）
- ✅ 符合業界最佳實踐
- ✅ 團隊協作友好

**缺點**：
- ⚠️ 需要修改一次程式碼結構

---

### **方案 2：雲端同步（最方便）** ⭐⭐

**適用工具**：OneDrive、Google Drive、Dropbox、iCloud

**步驟**：
1. 將整個 `E:\SideProject` 移動到雲端同步資料夾
2. 在家裡電腦安裝同步軟體
3. 自動同步所有文件

**優點**：
- ✅ 完全自動化
- ✅ 包含所有文件
- ✅ 跨裝置即時同步

**缺點**：
- ⚠️ 敏感資訊存在雲端（建議啟用加密）

**建議設置**：
```
# 資料夾結構
OneDrive/
  └── Dev/
      └── SideProject/  ← 整個專案放這裡
```

---

### **方案 3：加密打包（手動但安全）** ⭐

**適用場景**：偶爾換電腦，不需頻繁同步

**步驟**：

#### **當前電腦打包**
```bash
cd e:\SideProject\01-開發實驗\程式碼
tar -czf sensitive.tar.gz *.gs *.csv
```

#### **傳輸方式（任選一種）**
- 🔐 加密 USB 隨身碟
- 📧 加密郵件（7-Zip 加密壓縮）
- 💬 私人通訊軟體（Telegram 私密對話）

#### **家裡電腦解壓**
```bash
cd C:\...\SideProject\01-開發實驗\程式碼
tar -xzf sensitive.tar.gz
```

**優點**：
- ✅ 完全控制傳輸過程
- ✅ 可加密保護

**缺點**：
- ⚠️ 需要手動操作
- ⚠️ 容易忘記更新

---

### **方案 4：使用 Git 的 git-crypt（進階）** ⭐⭐⭐

**原理**：加密特定文件後再推送到 Git

**步驟**：
1. 安裝 `git-crypt`
2. 設定加密規則
3. 推送時自動加密，拉取時自動解密

**優點**：
- ✅ 完全自動化
- ✅ 加密存儲在 Git
- ✅ 業界標準方案

**缺點**：
- ⚠️ 需要安裝額外工具
- ⚠️ 設置稍複雜

**參考文件**：https://github.com/AGWA/git-crypt

---

## 💡 我的建議

### **如果你只有 2 台電腦**
→ 選擇 **方案 2：雲端同步**（最簡單）

### **如果重視安全性**
→ 選擇 **方案 1：配置文件分離**（最佳實踐）

### **如果偶爾換電腦**
→ 選擇 **方案 3：加密打包**（彈性最大）

## 🛠️ 立即執行：方案 2（雲端同步）

### **Step 1：移動專案到雲端資料夾**
```bash
# 假設你使用 OneDrive
move E:\SideProject C:\Users\YourName\OneDrive\Dev\SideProject
```

### **Step 2：更新 Git 工作目錄**
```bash
cd C:\Users\YourName\OneDrive\Dev\SideProject
git status  # 確認 Git 正常運作
```

### **Step 3：在家裡電腦**
1. 登入 OneDrive
2. 等待同步完成
3. 開啟 Cursor，載入專案
4. 完成！

## ⚠️ 安全提醒

無論選擇哪種方案，請確保：

1. ✅ 定期更新 LINE Token（建議 3-6 個月一次）
2. ✅ 不要將敏感文件分享給他人
3. ✅ 如果 Token 外洩，立即到 LINE Developers 重新生成
4. ✅ 使用雲端服務時，啟用兩步驟驗證（2FA）
