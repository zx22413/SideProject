# 🧪 實驗紀錄：模型表現與人格迭代 (Model & Persona Pivot)

> [!abstract] 核心發現
> **日期**：2025-12-30
> **關鍵轉折**：從 Linebot 開發轉向 GPTs 快速驗證 -> 發現 GPT-5.2 的文學偏誤 -> 誕生「史伊 2.0」人格。

## 1. 實驗背景 (Context)
* **原計畫**：直接串接 Linebot + Coze 進行開發。
* **遭遇問題**：開發成本高，需先驗證「對話體驗」是否吸引人。
* **調整策略**：先利用 ChatGPT (GPTs) 進行無代碼測試 (Concierge MVP)。

## 2. 模型對決：GPT-4o vs GPT-5.2 (The Model Clash)
在測試「吃燒肉便當」與「Day 2 心情」時，發現了顯著差異：

| 比較維度 | GPT-4o (The Actor) | GPT-5.2 (The Scholar) |
| :--- | :--- | :--- |
| **預設風格** | **鬆弛、口語化** | **嚴謹、文學化、散文感** |
| **對「描述」的理解** | 用驚嘆號、斷句表達情緒。<br>Ex: "哇！是燒肉耶！" | 用修辭、比喻描寫意象。<br>Ex: "油香像低空的暖流..." |
| **指令服從度** | 容易扮演笨角色。 | 傾向於展現聰明，難以「降智」。 |
| **測試結論** | ✅ **勝出** (符合寵物感) | ❌ **失敗** (太像文青作家，不像寵物) |

## 3. 解決方案：史伊 2.0 (Sui-Style Pivot)
為了在免費版 (強制使用 5.2/4o-mini) 也能達成 4o 的呆萌感，單純的「禁止寫小說」指令無效。我們採用了 **「極端人格模仿」** 策略。

* **舊 Prompt**：「禁止文學感、口語化、30字以內。」
    * *結果*：模型產生邏輯衝突，變成破碎的詩句。
* **新 Prompt (史伊 2.0)**：「模仿《異世界流浪美食家》的**史伊 (Sui)**。使用幼兒詞彙、疊字、直接行動。」
    * *結果*：成功強迫 GPT-5.2 放棄高智商詞彙，轉為 "行為導向" (啊姆啊姆、噗呦)。

## 4. 結論 (Takeaway)
1.  **Prompt 不能只給限制**：只叫模型「不要做什麼」(Don't) 效果很差。
2.  **要給具體模仿對象**：直接指定 "史伊 (Sui)" 這種強烈風格的角色 (Do)，模型能瞬間抓到語感。
3.  **產品定位修正**：從「懂你的雲端伴侶」調整為「呆萌貪吃的史萊姆雲」。
