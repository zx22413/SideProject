# Daily Journal Sync Skill

## 觸發時機
- 用戶**明確執行** `/sync-journal` 或 `/archive-old-journals` 時。
- 用戶在**編輯或完成** `00-工作區/工作報告_*.md` 並詢問同步時。
- **不觸發**：用戶正在編輯 .gs 或 `01-開發實驗/程式碼/` 時不主動調用；工作進度與程式碼撰寫分離。

## /sync-journal 指令邏輯 (執行優先)
1. **讀取**：抓取 `00-工作區/` 下最新的一份 `工作報告_*.md`。
2. **分析**：識別日記中 `## 🔗 受影響的核心文件` 區塊下的所有連結。
3. **跨文件同步**：
   - **README.md**：若有專案結構或版本進度變更，同步更新根目錄 `README.md` 的「專案結構」「專案進度」區塊。
   - **企劃書**：更新版本號與開發進度（參考 .gs 檔頭）。
   - **Backlog**：根據日記內容勾選 [x] 已完成事項。
   - **對照表**：若有新變數（如 userState 新欄位），同步映射關係。
   - **決策鏈**：將日記中帶有 `[!important]` 的決策點搬運。
4. **驗證**：確保所有產出的連結均符合 `[[相對路徑]]`。
5. **提示**：同步後強制提示用戶「已止血完成，建議重置對話 (Cmd+N)」。

## /archive-old-journals 指令邏輯
1. 掃描 `00-工作區/` 下超過 7 天的日記。
2. 根據年份與月份建立或識別 `archive-YYYY-MM/` 資料夾。
3. 移動檔案並更新相關 Tier 1 文件的連結引用。

## 限制規範
- **禁止全文複製**：僅搬運關鍵結論，維持 Tier 1 文件精簡。
- **語法依賴**：調用官方 `obsidian-markdown` 處理 Callouts 格式。
