### 2. 解決五大設計問題

#### 問題 A：黑貓說教感太重
- **原因**：黑貓像「新手教學 NPC」，直接解釋世界觀和玩法
- **解決**：黑貓變成「神秘同事」，只給簡短提示，不解釋
- **範例**：
  ```
  ❌ 舊版：「你是罪惡吞食者，工作是用料理喚醒靈魂記憶...」
  ✅ 新版：「會有客人來。煮點吃的給他們。」
  ```

#### 問題 B：「罪惡吞食者」太中二
- **原因**：詞彙過於奇幻，與樸實的故事調性不符
- **解決**：完全去除中二詞彙，改用日常語言
- **刪除詞彙**：罪惡吞食者、CONFITEOR、罪名吞噬者、靈魂
- **保留概念**：主廚、料理人、食堂、客人、迷途的人

#### 問題 C：食材解鎖機制不自然
- **原問題**：食材直接給予，缺乏探索感
- **新設計**：**記憶即食材系統**
  ```
  對話中：
  田中：「她笑起來...像蜜糖...」
  
  系統自動：
  [記憶碎片：蜜糖般的笑容] 已收集
  → 廚房解鎖：🍯 蜂蜜（抽象食材）
  
  料理時：
  「你將金色的笑容記憶放入鍋中...」
  ```

#### 問題 D：缺乏儀式化結構
- **原問題**：畫鬼腳太直線，缺乏探索節奏
- **新設計**：參考紅弦俱樂部 + 逆轉裁判
  ```
  🌙 Night Shift（觀察期）→ 點擊互動點，獲取線索
  🌅 Day Shift（對話期）→ 選擇話題，套取情報
  🍳 Cooking Time（料理期）→ 用記憶食材做菜
  🌃 After Hours（揭露期）→ 記憶劇場觸發
  ```

#### 問題 E：第一章缺乏神秘感
- **核心衝突**：
  - 喜歡「引子」（黑貓介紹世界觀）
  - 但想要「田中不知道自己已死」的懸疑
  - 兩者會互相破壞
- **解決方案**：提供兩種開場選項
  - **方案 A**：完全神秘（玩家也不知道真相）
  - **方案 B**：倒敘詩意（「後來我才知道...」框架）

### 4. 設計「記憶即食材」系統

**核心概念**：「你用他的記憶，做出他的料理」

**解鎖流程**：
```
對話 → 關鍵詞出現 → 系統捕捉 → 食材解鎖 → 料理使用
```

**食材分類**：
- **基礎食材**：水、米、蔬菜（物理食材）
- **記憶食材**：💧寒冷、🪡針、🍯蜜糖笑容（抽象食材，發光顯示）

**烹飪邏輯**：
- 將發光的記憶食材「放入鍋中」
- 記憶在高溫下融化，裹住料理
- 料理變成「記憶的容器」
- 靈魂吃下 = 記憶回歸

## 💡 今日學習 / 經驗

### 1. 敘事設計的「說教陷阱」
- **問題**：太想解釋世界觀，結果破壞沉浸感
- **解決**：Show, don't tell —— 透過體驗揭露，而非直接說明
- **範例**：黑貓不說「這是生死交界」，而是「會有客人來」

### 2. 「記憶即食材」的設計洞察
- **核心**：料理不只是機制，是「情感的煉金術」
- **邏輯**：
  ```
  靈魂說出記憶 → 記憶變成食材 → 做成料理 → 還給靈魂
  = 完整的情感循環
  ```
- **參考**：Venba（料理喚起記憶）+ 紅弦俱樂部（物品即情報）

### 3. 儀式化設計的節奏感
- **觀察**：紅弦俱樂部和逆轉裁判都有「日常→高潮」的節奏
- **應用**：
  ```
  觀察（低壓）→ 對話（中壓）→ 料理（高壓）→ 劇場（釋放）
  ```
- **目的**：避免單調，製造情感起伏

### 4. 開場的「神秘感 vs 引子」困境
- **困境**：想要神秘，又想要詩意開場
- **解法**：倒敘框架 —— 「後來我才知道...但那天...」
- **效果**：保留兩者，製造期待感
