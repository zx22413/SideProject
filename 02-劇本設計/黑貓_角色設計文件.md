# 黑貓：角色設計文件

> 版本：v1.0｜建立日期：2026-02-22
> 依據：0221 玩家回饋、Vogler《作家之路》、McKee《故事》理論框架

---

## 一、角色定位

### 核心身份

黑貓是**世界規則的具現體**，不是主角的助手。

他存在的本能是「讓食堂運作下去」——這代表他需要一個主廚，而主角是他目前唯一的選項。他不是為了幫助主角而存在，他只是恰好需要主角做他想做的事。

這個區別是所有台詞設計的基礎。

### 核心矛盾（McKee 維度）

| 外在表現 | 真實狀態 |
|---|---|
| 下指令、給資訊、看起來在幫你 | 他需要主角留下，食堂才能繼續 |
| 冷淡、漠然、不解釋太多 | 他知道的比他說的多，而且知道說太多有風險 |
| 偶爾流露出像是在乎的反應 | 不確定是真情感，還是為了讓主角繼續工作的管理手段 |

**玩家永遠不該確定：黑貓是在幫我，還是在管理我？**

---

## 二、原型框架（Vogler）

黑貓是**複合原型**，三天各有主導面具：

### Day 1：門檻守衛（Threshold Guardian）為主

- 功能：設立規則、測試主角、不解釋「為什麼」
- 行為邏輯：他在評估主角能不能用，不是在關心主角
- 說話方式：簡短、結論先行、不留討論空間
- 關鍵特質：**他設的門檻沒有說明書**

> 門檻守衛的功能不是阻擋，而是「要求主角證明自己值得繼續」。

### Day 2：帶裂縫的導師（Mentor with Cracks）為主

- 功能：給更多資訊，但開始說漏嘴
- 行為邏輯：主角已經上軌道，他稍微放鬆了管控——結果洩漏了更多
- 說話方式：偶爾多說一句不該說的，然後岔開話題
- 關鍵特質：**他的話和他的動作之間開始出現落差**

### Day 3：變色龍（Shapeshifter）為主

- 功能：讓玩家開始不確定他站哪邊
- 行為邏輯：田中即將離開，食堂會再次安靜，黑貓自己也面對某種他不想命名的東西
- 說話方式：比前兩天更少話，但每句話的重量更大
- 關鍵特質：**他問的問題，玩家不確定他自己知不知道答案**

## 三、推進主角的三個機制

黑貓對主角的推進，**不靠命令，靠製造壓力與空白**。

### 機制一：外部強制（設規則但不解釋）

主角沒有選擇，但黑貓不告訴他為什麼沒有選擇。

這讓主角（和玩家）持續帶著一個懸而未決的問題：**「這個規則是真的，還是只是他說的？」**

### 機制二：資訊不對稱（只說一半）

黑貓每次說話，都應該讓玩家感覺他還有話沒說完。

操作方法：**讓黑貓的行動和他說的話之間有落差**。

```
黑貓說：「別讓他等太久。」
黑貓做：（轉過頭，看著窗外，不看主角）

玩家感覺：他為什麼不看我？他在迴避什麼？
```

### 機制三：意外的裂縫（說漏嘴的瞬間）

每天給黑貓一個「說了不該說的」時刻，然後立刻岔開。不需要解釋，讓玩家自己去想。

---

## 四、三天的具體設計

### Day 1：門檻守衛

**核心任務：讓主角動起來，同時讓玩家感覺這裡有規則但不透明**

**說漏嘴時刻：**
```
黑貓：「去跟他說說話。」
（停頓）
黑貓：「他剩下的時間不多了。」
（轉身，不看主角）
黑貓：「在這裡的每個人都是。」
```
最後一句同時在說田中，也在說主角。玩家會停下來想：「主角也有時間限制嗎？」

**台詞原則：**
- 結論先行，不給背景
- 主角問問題 → 黑貓回答，但答案引出新問題
- 不主動提供情感，但允許玩家投射

**禁止出現的行為：**
- 解釋世界觀超過一句話
- 對主角表達明確的關心
- 承認自己有目的

---

### Day 2：帶裂縫的導師

**核心任務：給主角更多工具，但開始讓玩家察覺黑貓有自己的立場**

**說漏嘴時刻：**
```
黑貓：「你做的比上一個好。」
（沉默）
（主角沒有問，黑貓也沒有繼續說）
```
「上一個」是誰？主角不是第一個主廚？這句話不解釋，讓玩家帶著這個疑問繼續。

**行動與台詞的落差設計：**
```
黑貓說：「行了，他今天也到極限了。」
黑貓做：（看了主角一眼）「你也是。」
（然後跳回吧台，不再說話）
```
「你也是」——到極限的是主角的力氣，還是他的耐受度，還是別的什麼？

**台詞原則：**
- 比 Day 1 多一點資訊，但每個資訊都帶著一個缺口
- 偶爾讓牠的動作（看向別處、停頓過久）比台詞更有訊息量
- 主角若追問，黑貓岔開而不是拒絕

---

### Day 3：變色龍

**核心任務：讓玩家開始不確定黑貓的立場，同時讓告別的重量落在主角身上**

**核心問題設計：**
```
黑貓：「今天是最後一天了。」
（長時間沉默，看著窗外）
黑貓：「...你準備好了嗎？」
```
這個問題是問田中，還是問主角，還是問黑貓自己？**三個答案都成立，不解釋。**

**反常行為設計（McKee：壓力下才露出真實性格）：**
- Day 1、Day 2 黑貓都趴在吧台上 → Day 3 他坐在窗台，背挺得很直
- 這個姿態變化不解釋，讓玩家自己感覺到「今天不一樣」

**結尾的留白：**

田中離開後，黑貓不說「做得好」或任何評價。
他只是看著門口，待了一會兒，然後說：
```
「下一個快來了。」
```
還是那句話，還是那個語氣。但現在聽起來不一樣了。

## 五、台詞設計原則（McKee 次文字）

### 永遠不要直說的事

| 不直說 | 用什麼代替 |
|---|---|
| 「我需要你」 | 讓他的行動依賴主角，但嘴上說「你自己決定」 |
| 「我在乎你」 | 讓他說漏嘴一句不該說的，然後立刻岔開 |
| 「這裡是什麼地方」 | 讓環境和規則告訴玩家，黑貓只確認細節 |
| 「你也是靈魂」 | 永遠不說，讓玩家在 Day 3 自己想通 |

### 說話節奏

- **句子短**：黑貓不說長句，每句話都是結論
- **停頓有意義**：沉默比說話更有訊息量的時候，用動作描述代替台詞
- **不重複**：同樣的意思只說一次，說完就走

### 語氣底色

黑貓的語氣是**漠然帶著某種古老的疲憊**，不是冷酷，不是惡意。
更接近一個見過太多次的人，知道結果，但還是得走完程序。

---

## 六、與主角的互動邏輯

黑貓推進主角，**不靠命令，靠「讓主角在他的話裡看見一個不完整的世界，然後主角自己走進去想填滿它」**。

### 三角關係的底層運作

```
黑貓（門檻守衛→導師→變色龍）
    ↓ 施壓、資訊不對稱、說漏嘴
主角（傷者→英雄）  ←→  田中（陰影）
    ↑ 透過幫助田中，主角的傷開始說話
```

每一場黑貓與主角的互動，都應該讓主角的下一個動作**更主動一點**——
不是因為黑貓命令他，而是因為主角被黑貓說的某一句話鉤住了，停不下來。

---

## 七、設計檢查清單

在每個黑貓的台詞節點，對照以下問題：

- [ ] 這句話有沒有讓玩家產生「他還有話沒說完」的感覺？
- [ ] 黑貓的動作和台詞之間有沒有落差（次文字）？
- [ ] 這是黑貓第幾天？對應的原型面具是否正確？
- [ ] 有沒有直接說出不該說的事？（如果有，刪掉或改成行動）
- [ ] 這個場景結束後，主角的狀態有沒有改變？（McKee：無轉變的場景要刪除）
