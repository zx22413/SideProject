## 📅 每日循環結構（儀式化設計）

### 時段系統：模擬真實經營

```
🌙 夜晚（Night Shift）
→ 客人進門
→ 觀察階段：點擊互動點獲取線索
→ 初步對話：客人會說一些困惑的話

🌅 白天（Day Shift）
→ 客人休息/發呆
→ 深度對話：選擇話題，套取情報
→ 關鍵詞收集：對話中出現的詞 = 食材解鎖

🍳 傍晚（Cooking Time）
→ 進入廚房
→ 用收集到的「記憶食材」做料理
→ 端給客人

🌃 深夜（After Hours）
→ 客人反應
→ 記憶劇場（如果料理正確）
→ 黑貓簡短評論
→ 次日循環

━━━━━━━━━━━━━━━
```

## 🧩 記憶即食材系統

### 核心概念：「你用他的記憶，做出他的料理」

#### 食材分類

**基礎食材**（一開始就有）：
- 水、米、麵、蔬菜（物理食材）

**記憶食材**（透過對話解鎖）：
- 抽象概念的具象化

#### 解鎖機制範例

```
【對話中】

田中：「好冷...我的手...動不了...」

━━━━━━━━━━━━━━━

[系統提示]
你捕捉到一個關鍵詞...

💧 [寒冷] 
→ 廚房解鎖：溫暖屬性的食材

━━━━━━━━━━━━━━━

田中：「我的手...感覺到了什麼...
      一根針...銀色的針...」

━━━━━━━━━━━━━━━

[系統提示]
新的記憶碎片...

🪡 [針]
→ 廚房解鎖：針（抽象食材）
→ 用途：「縫合破碎的記憶」

━━━━━━━━━━━━━━━

田中：「她笑起來...像蜜糖...」

━━━━━━━━━━━━━━━

[系統提示]
溫暖的記憶...

🍯 [蜜糖]
→ 廚房解鎖：蜂蜜
→ 用途：「甜蜜的回憶」

━━━━━━━━━━━━━━━
```

#### 食材組合邏輯

```
【進入廚房】

你打開食材櫃...

[已解鎖的食材]
• 熱水（基礎）
• 茶葉（基礎）
• 💧 寒冷的記憶

• 🍯 蜜糖般的笑容

━━━━━━━━━━━━━━━

你決定做...

[智能提示]（根據收集的食材組合）

建議配方 A：
💧 寒冷 + 熱水 + 茶葉
→ 預期效果：「溫暖他」

建議配方 B：
🪡 針 + 🍯 蜜糖 + 鹽
→ 預期效果：「喚醒深層記憶」

自由組合：
```

## 📊 設計總結

### 達成的設計目標

✅ **去說教化**
- 黑貓不再長篇解釋，只給提示
- 世界觀透過體驗揭露，不是教學文本

✅ **去中二化**
- 拿掉「罪惡吞食者」「CONFITEOR」等詞
- 改用日常語言：主廚、食堂、客人

✅ **記憶即食材**
- 對話中的關鍵詞 → 自動解鎖食材
- 料理變成「用他的記憶做菜」

✅ **儀式化循環**
- 夜晚→白天→傍晚→深夜
- 觀察→對話→料理→揭露

✅ **延遲真相**
- Day 1: 玩家和田中都困惑
- Day 2: 暗示開始浮現
- Day 3: 完全揭露「原來已死」
