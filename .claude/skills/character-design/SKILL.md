# Character Design（角色設計）

> 理論來源：Robert McKee《故事》/ Christopher Vogler《作家之路》

## When to Use

- 設計新客人角色（Guest N）的核心人格與背景
- 判斷某個角色「感覺扁平、沒有厚度」時診斷
- 規劃角色的維度矛盾與在壓力下的真實反應
- 設計食罪者（玩家）與配角的原型功能

---

## 核心區分：特徵 vs 真實人格

麥基最重要的概念——**角色特徵不等於角色人格**。

| | 人物特徵（Characterization） | 真實人物（True Character） |
|---|---|---|
| **定義** | 所有可觀察的外在特質 | 在極大壓力下做出的選擇 |
| **包含** | 年齡、職業、口音、習慣、外貌 | 恐懼、慾望、道德核心 |
| **如何揭示** | 描述即可呈現 | 只能透過衝突與選擇揭示 |
| **陷阱** | 寫了很多細節但角色仍然空洞 | 沒有讓角色面對真正的抉擇 |

**麥基的校車比喻**：在一輛失火校車旁，名醫和無證清潔工——誰會衝進去救人？特徵完全相反，但**在極限壓力下的行動**才揭示真實人格。當外在特徵與真實人格形成反差時，故事產生深度。

**靈魂食堂應用**：
- 田中太郎的特徵：沉默老人、日本口音、整齊的衣物
- 真實人格：只有在「說出那個名字」（最高壓力時刻）才能被揭示

> 設計時問：「這個客人在 Day 3 最後的抉擇時，他真正的恐懼是什麼？」

## 角色維度（Dimension）：矛盾 = 深度

麥基將角色「維度」定義為**內部的矛盾力量**。

一維角色：在所有情境下展現一致特質 → 乏味、可預測

多維角色：核心人格中藏著相互衝突的力量

| 設計方法 | 說明 | 例子 |
|---------|------|------|
| **對立特質並存** | 同一人物同時擁有兩個看似矛盾的特質 | 慈父 × 冷酷生意人 |
| **情境切換** | 在不同情境下展現截然不同的一面 | 對陌生人慷慨 × 對家人吝嗇情感 |
| **自我認知偏差** | 角色對自己的認知與外人看到的不同 | 以為自己無私，其實是控制慾 |

**新客人設計工作表（維度）**：

```
表面特質：___（外人第一眼看到的）
深層矛盾：___ × ___（兩個相互衝突的內在力量）
自我認知偏差：他以為自己是___，但其實是___
觸發維度翻轉的時刻：當___發生時，他的另一面才會出現
```

## 星系模型：群戲設計

麥基的星系模型——**主人公是太陽，配角的存在是為了照亮主人公的不同維度**。

```
        主人公（太陽）
       /      |      \
   角色A    角色B    角色C
（激發勇氣）（引誘貪婪）（照見恐懼）
```

**靈魂食堂的星系結構**：

| 角色 | 原型功能 | 照亮的維度 |
|------|---------|-----------|
| **客人**（如田中太郎） | 主人公（旅程的中心） | 執念的根源與轉化 |
| **食罪者（玩家）** | 導師 + 盟友 | 引導客人面對深層真相 |
| **黑貓** | 使者 + 惡作劇者 | 世界規則的解說、打破嚴肅 |
| **其他靈魂的殘影** | 陰影 + 門檻守衛 | 客人未解決的恐懼的具象化 |

**設計原則**：話題選項中每個話題方向，應觸發客人的不同維度。不要讓所有話題都往同一個情緒方向走。

## Vogler 八大原型：功能性角色設計

原型是**戲劇功能**，不是固定角色身分。同一個角色可以在不同階段扮演不同原型。

| 原型 | 心理功能 | 戲劇功能 | 靈魂食堂應用 |
|------|---------|---------|------------|
| **英雄（Hero）** | 自我尋求完整 | 觀眾的情感投射，在旅程中學習 | 客人（自身救贖旅程的中心） |
| **導師（Mentor）** | 智慧、良知的具象 | 提供訓練、建議、關鍵禮物 | 食罪者（玩家）；料理本身 |
| **門檻守衛（Threshold Guardian）** | 內在恐懼、防衛機制 | 在轉折點設置障礙 | 客人的迴避行為；未說出的名字 |
| **使者（Herald）** | 對改變的渴望 | 宣告冒險開始 | 黑貓；食堂本身的出現 |
| **變色龍（Shapeshifter）** | 不穩定、難以捉摸 | 增加懸疑，挑戰判斷力 | 客人在不同話題下展現的矛盾面 |
| **陰影（Shadow）** | 被壓抑的力量、負面情緒 | 主要對手，逼迫成長 | 客人自身的執念；未解的悔恨 |
| **盟友（Ally）** | 社交支持系統 | 陪伴、分擔、提供新視角 | 食罪者在某些支線的角色 |
| **惡作劇者（Trickster）** | 破壞現狀的本能 | 緩解張力，指出荒謬 | 黑貓；某些話題的意外方向 |

**設計提醒**：
- 導師不一定是正面的（黑暗導師、墮落導師）
- 陰影給予人性化的弱點與合理動機，才能引起共鳴
- 客人在某個時刻可以從英雄變為門檻守衛（拒絕前進）

## 對抗力量三層次：衝突設計

一個有深度的衝突必須同時觸及三個層次：

| 層次 | 範疇 | 診斷問題 |
|------|------|---------|
| **內部衝突** | 心智、身體、情感 | 客人自己與自己的掙扎是什麼？ |
| **個人衝突** | 親友、戀人、家人 | 客人與具體的人之間有什麼未竟之事？ |
| **超個人衝突** | 社會、時代、命運 | 什麼外部力量（時代、文化、命運）造成了這個困局？ |

**如果某個話題只有個人衝突（事件層）而沒有內部衝突（心理層），角色感覺淺薄。**

**田中太郎範例**：
- 內部：無法原諒自己當年的沉默
- 個人：與兒子之間從未說出口的話
- 超個人：戰後日人在台灣的認同困境

## 四角價值矩陣：結局設計深度

麥基要求：好的故事必須探索到「負面之負面」，不只是在正向與對立之間徘徊。

| 層次 | 定義 | 客人執念範例（「愛」） |
|------|------|-------------------|
| **正向** | 故事追求的終極目標 | 真正的和解與被接受 |
| **相反** | 灰色地帶 | 疏離（冷漠地共存） |
| **矛盾** | 直接對立 | 恨（主動的傷害） |
| **負面之負面** | 偽善下的極致惡 | 自我憎恨（把愛的對象當成自我毀滅的手段） |

**設計客人執念時用此矩陣**：
1. 這個客人的正向目標是什麼？
2. 他現在處於哪個層次？（通常是「相反」或「矛盾」）
3. BITTER 結局應把他推向「負面之負面」
4. SWEET 結局是他真正抵達「正向」

## 新客人設計工作表

設計一個新靈魂客人時，依序回答：

### 一、外在特徵（Characterization）
```
姓名：___
時代背景：___（什麼年代的人？）
職業/身分：___
外在特質3個：___、___、___
口語習慣/說話方式：___
第一次出現在食堂的樣子：___
```

### 二、真實人格（True Character）
```
執念的根源：___（發生了什麼事？）
從未說出口的話：___（對誰？說什麼？）
最深的恐懼：___（比執念更深的東西）
在最高壓力下會做什麼選擇：___
表面特徵 × 深層人格的反差：___
```

### 三、角色維度（Dimension）
```
表面上他是：___
但實際上他是：___（對立的內在力量）
哪個話題會觸發維度翻轉：___
他不願承認的自己的哪一面：___
```

### 四、原型功能分配
```
主原型：英雄（固定）
對玩家而言，他何時是「門檻守衛」：___（拒絕推進的時刻）
他何時是「陰影」：___（讓玩家/食罪者感到挑戰的時刻）
他何時是「盟友」：___（如果有的話）
```

### 五、三層衝突設計
```
內部衝突：___（心理/情感層的自我鬥爭）
個人衝突：___（與具體的誰，有什麼未竟之事）
超個人衝突：___（時代/文化/命運的力量）
```

### 六、四角價值矩陣
```
他追求的正向價值：___
他目前處於的狀態（相反/矛盾）：___
BITTER 結局的「負面之負面」：___
SWEET 結局的真正和解：___
```

### 七、三天弧線草圖
```
Day 1 的防衛姿態（拒絕召喚）：___
Day 2 深掘的核心話題：___
Day 3 最終的兩難抉擇：___（兩個同等分量的選擇）
```

## 潛文本設計原則

> 「若場景就是關於場景本身，那你就有麻煩了。」——麥基

**潛文本（Subtext）**：角色說的話永遠不等於他真正的意思。

| 表面文本（Text） | 潛文本（Subtext） |
|----------------|-----------------|
| 「那時候天氣很冷。」 | 「那一天我犯了一個永遠無法彌補的錯。」 |
| 「我不在乎了。」 | 「我在乎，但承認在乎讓我無法承受。」 |
| 「你做的魚很好吃。」 | 「這讓我想起了我失去的某個人。」 |

**設計對話時**：先寫潛文本，再寫表面文本。讓角色透過看似無關的話題、物件或習慣性動作傳達真實情感。
