# GPTs (技術架構)

> [!info] 快速連結
> 詳細技術架構請見：[[MVP 企劃：7日陪伴實驗_V1.2#2. 產品規格 (MVP Specs)]]

## 快速摘要

**GPTs** 是 [[MVP 企劃：7日陪伴實驗]] 的技術架構核心，用於實現 [[雲寶]] 的對話功能。

### 技術定位

- **前端**：GPTs（ChatGPT 的自訂 GPT 功能）
- **後端**：無（純 GPTs，無需額外開發）
- **推播**：[[人肉推播]]（LINE 私訊/群組）

### 為什麼選擇 GPTs？

- ✅ **零開發成本**：不需要寫程式，只需要設定 Prompt
- ✅ **快速驗證**：可以立即開始測試，不需要等待技術開發
- ✅ **靈活調整**：可以隨時修改 Prompt，優化對話體驗
- ✅ **符合 MVP 目標**：重點是驗證「對話體驗」是否吸引人，而非技術實現

### 核心設定

**Prompt 來源**：[[角色設定：雲寶#3. 靈魂腳本 (System Prompt)]]

**主要功能**：
1. **角色扮演**：扮演 [[雲寶]]，使用史伊式說話法則
2. **情緒代謝**：把用戶的話當成食物，用史伊語氣反應
3. **多選題模式**：處理 DayX A/B 選擇題，建立記憶標籤
4. **進化與結局**：根據選擇題 + 對話判斷 Day 3 進化、Day 7 結局
5. **好奇寶寶模式**：每次回覆都拋出一個小問題，引導繼續對話

### 使用流程

1. **用戶收到推播**：LINE 推播提醒「今日選擇題」
2. **點擊連結**：進入 GPTs 對話窗
3. **輸入選擇**：輸入「DayX A」或「DayX B」
4. **GPT 回應**：雲寶用史伊語氣反應，並延伸問題
5. **繼續對話**：用戶可以繼續分享情緒，雲寶會記住並回應

### 限制與未來規劃

**V1.2 限制**：
- ❌ 無法看到完整對話 log（GPTs 不提供後台）
- ❌ 無法自動判定進化/結局（需要手動判斷）
- ❌ 無法自動推播（需要人肉推播）

**V2.0 規劃**：
- 開發 Line 官方帳號，串接 Coze/API
- 實現自動推播、後台 log、自動判定進化/結局
