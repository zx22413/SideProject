## 🎯 專案核心概念

### 世界觀
玩家經營一間位於「生與死交界」的食堂，接待迷途的靈魂。這些靈魂因執念而困在時間迴圈中，無法前往來世。

### 玩家目標
透過對話收集靈魂的記憶線索，找出他們生前最懷念的料理，烹飪並呈上。當靈魂吃到喚起記憶的料理後，執念解開，可以安心離去。

### 核心機制
1. **Day 1（感官線索）**：透過互動收集感官標籤（溫暖、柔軟、鹹味等）
2. **Day 2（記憶拼圖）**：標籤組合觸發記憶場景，解鎖目標料理
3. **Day 3（料理超渡）**：烹飪料理 → 記憶重現 → 告別儀式 → 獲得遺物

---

## 🔑 設計核心原則

### 三大不變的底層規則（參考 [[../開發決策鏈]]）

1. **低成本、非同步互動**
   - 採用回合制，玩家主動觸發 → Reply API 免費
   - 避免即時推播 → 零推播成本

2. **高情感投入**
   - 3 日濃縮 Bird Alone 的情感曲線
   - 每日都有高峰時刻（收集線索 → 解鎖記憶 → 完成告別）

3. **文字作為核心媒介**
   - 玩家的文字輸入轉化為「抽象食材」
   - 對話選擇影響劇情分支

## 🚨 與舊版本的關鍵差異

| 維度 | 雲寶（V1.0-V1.3） | 靈魂食堂（V1.5） |
|------|------------------|------------------|
| **定位** | 電子寵物 + 日記 | 敘事遊戲 |
| **時長** | 7 日 | 3 日 |
| **角色** | 單一寵物（雲寶） | 多位靈魂（Guest 系統） |
| **互動** | 餵食 + 聊天 | 解謎 + 料理 + 告別 |
| **目標** | 陪伴養成 | 救贖超渡 |
| **重玩性** | 低（劇情重複） | 高（每位靈魂不同） |
